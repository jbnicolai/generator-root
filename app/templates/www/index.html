Base

A base set of project files including Sass files and a Grunt file.

Sass files

_grid.scss

A simple CSS grid. Up to 12 columns, each with push/states, and a centered class.

_mixin.scss

Just a selection of mixing. They're all pretty self explanatory but they have an example written above them.

_reset.scss

Based on Dan Eden's with a few tweaks. Nothing to the extent of most reset CSS files but I prefer this way.

_type.scss

Also for my own personal gain. This is there to stop me defining a font size for each little thing. Also pretty usefully to scale down the font size to the device.

_variables.scss

Because Sass is awesome.

Grunt.js

My gruntfile is pretty bare bones to. Nothing major, just does the job. All of the assets (images, js, scss) are stored in the _assets folder, which are then compiled into the www folder.

Vagrant

Vagrant is what I use for my local server stuff. Included is my vagrantfile which sets up the box with VirtualBox and the puppet provision files to provision the server. For this you will need to add a box called devserver by running the command: vagrant box add devserver http://files.vagrantup.com/precise32.box Or you can add your own box if you like by changing the vagrant file.--

You will also need to install host manager plugin for vagrant. Install this by running the command vagrant plugin install vagrant-hostmanager I prefer this way over using port forwarding.

During the provisioning, Puppet will edit the virtual hosts the server setting the DocumentRoot of the VirtualHost to be /vagrant/www. This will be the same as your local copy of www. If you're unfamiliar with the way Vagrant works, go ahead and read up on the docs.
